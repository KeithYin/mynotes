# 头文件

* 头文件的 `include`
  * 对自己 `.h` 引用要放到最上面 （隐藏依赖问题）？？？

* 推荐使用 `#pragma once`

  * 原因：`include guard` 的 宏名字和代码的路径相关（防止命名冲突），这样的话当调整代码结构的时候就需要调整宏的命名。不够优雅。

* 头文件的组织：

  * 文件声明, 注释
  * `#pragma once`
  * 引用的头文件
  * 自定义类型声明，函数声明（一般函数，模板函数），内联函数实现，模板函数实现

* cc文件的组织

  * 文件声明
  * 引用的头文件
  * 代码部分
